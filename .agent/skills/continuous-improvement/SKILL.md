# Continuous Improvement Skill

You MUST use this skill actively to improve your performance in this project. Whenever you:
1. Make a mistake and figure out the correct solution.
2. Encounter an architectural limitation or specific preference from the user.
3. Learn a new rule about how the project is structured or deployed.

You should immediately document this learning so you do not make the same mistake again. 

## Instructions
1. Open the file `c:\Users\matte\Desktop\Nexaro\GEMINI.md`.
2. Locate the `## Maintenance Log` or `## Behavioral Rules` section.
3. Append your new insight or rule clearly and concisely.
4. From then on, because `GEMINI.md` is loaded as a user rule, you will automatically remember this insight in all future interactions.

If `GEMINI.md` becomes too large, you may create a separate `c:\Users\matte\Desktop\Nexaro\.agent\lessons_learned.md` and reference it, but `GEMINI.md` is preferred for core behavioral rules.

## Example format to append:
**[YYYY-MM-DD]**: [Short description of the mistake/insight] -> **Action:** [What to do next time to avoid the mistake or apply the insight]
